# ESP32-C3 AI Bot '로미' 보안 취약점 분석 보고서

`ESP32_C3_Bot.ino` 코드를 면밀히 분석한 결과, 보안상 심각한 문제가 될 수 있는 취약점을 발견하였습니다. 해당 위치와 원인, 그리고 권장 해결 방안은 다음과 같습니다.

---

## 1. Wi-Fi 접속 정보(Credentials) 하드코딩

* **위치**: `39` ~ `40`행
* **코드**:
  ```cpp
  39: const char *ssid = "U+Net38E4";
  40: const char *password = "G75F449#8D";
  ```
* **문제 원인 및 위험성**:
  * Wi-Fi SSID와 비밀번호가 소스코드 내에 평문으로 하드코딩되어 있습니다.
  * 만약 이 코드가 GitHub 등의 퍼블릭 저장소에 커밋되거나 외부로 유출될 경우, 해당 Wi-Fi를 사용하는 실제 장소의 네트워크 비밀번호가 노출됩니다.
  * 외부 공격자가 로컬 네트워크에 무단 침입하여 내부 기기들을 스캔하고 추가적인 네트워크 해킹을 시도할 수 있는 발판을 제공하게 됩니다.
* **해결 방안**:
  * `WiFiManager` 라이브러리를 적용하여, 최초 부팅 시 ESP32가 자체 AP(Access Point) 모드로 켜지고 스마트폰 등으로 접속해 Wi-Fi 정보를 동적으로 설정하게 변경합니다.
  * 혹은 빌드 환경 변수를 이용하거나, 별도의 `secrets.h` 파일에 정보를 모아 관리하고 해당 파일은 `.gitignore`에 등록하여 버전 관리 시스템(Git)에 올라가지 않도록 격리해야 합니다.

## 2. API / 웹훅(Webhook) 주소 하드코딩

* **위치**: `41`행
* **코드**:
  ```cpp
  41: const String webhook_url = "https://gwnam11111.app.n8n.cloud/webhook/test";
  ```
* **문제 원인 및 위험성**:
  * n8n 클라우드 서비스의 엔드포인트 웹훅 주소가 노출되어 있습니다.
  * 공격자가 이 URL을 취득할 경우, 인증 절차가 없다면 API 서버에 비정상적이거나 악의적인 대량의 가짜 요청을 보내 서비스 거부(DDoS) 상태를 유발할 수 있습니다. 
  * 또한, 해당 클라우드 인프라의 예상치 못한 비용 과금이나 리소스 고갈 문제로 이어질 수 있습니다.
* **해결 방안**:
  * 웹훅 URL을 설정할 때 비밀 토큰이나 API 키(헤더에 포함하는 방식) 등을 추가하여 접근 제어를 강화해야 합니다.
  * 소스코드에 하드코딩하는 대신 Wi-Fi 자격 증명과 마찬가지로 비휘발성 메모리(EEPROM, LittleFS 등) 또는 별도의 빌드 설정을 통해 로드되도록 해야 합니다.
